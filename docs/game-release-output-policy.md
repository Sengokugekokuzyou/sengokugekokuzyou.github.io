# ゲーム更新出力ルール

## 通達

itch.ioへの自動アップロードは停止する。全ゲーム共通で、更新作業後はアップロード実行ではなく、手動確認用の出力物を作成する。

## 出力先

更新作業を行った場合は、次の2か所へ同じ最新版一式を出力する。

- `更新出力/最新フォルダ/`
- `C:\Users\kurai\Documents\外出先用`

`C:\Users\kurai\Documents\外出先用` には、最新版一式に加えて、itch.io開発ログへ貼り付けるためのアップデート情報を必ず入れる。

## 外出先フォルダに必ず入れるもの

- アップロード用ZIPまたはHTML5ゲーム一式
- バージョン番号
- リリース種別（本リリース / アップデート / テスト公開）
- 更新日
- 対象ゲーム名
- itch.io公開URL
- GitHubコミットURL
- 開発ログ用本文
- 手動確認チェックリスト

## Claude Codeへ直接指示する場合

Claude Codeへ直接作業を指示した場合も、このルールを適用する。

Claude Codeはitch.ioへ自動アップロードしない。更新物を `更新出力/最新フォルダ/` と `C:\Users\kurai\Documents\外出先用` に出力し、外出先用フォルダには `itch-devlog.md` を作成する。

## 開発ログ構成

itch.io開発ログは、以下の既存記事の構成をテンプレートにする。

https://sengokugekokujobeats.itch.io/kekkanokeifu/devlog/1539730/update-v20-retainers-council-debates-new-endings

基本構成:

1. 英語タイトル
2. 日本語本文
3. 今回の主な更新
4. 追加・改善された機能
5. バランス調整または不具合修正
6. プレイヤーへの案内
7. 英語要約
8. ファイル/バージョン情報

## 禁止事項

- itch.ioへ自動アップロードしない
- Discordへ未確認の公開告知を自動投稿しない
- 開発ログ本文なしで外出先フォルダを作らない
- 操作確認前のZIPを正式版として扱わない
