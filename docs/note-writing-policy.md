# note記事 執筆ルール

## 最優先方針

note記事は、情報の羅列よりも読者の共感を優先する。

読者が「自分にもこういう迷いがある」「この制作過程を追いかけたい」と感じられるように、制作側の気持ち、判断の背景、迷い、手触りを必ず入れる。

## 文体

- 説明だけで終わらせず、制作中に感じたことを入れる。
- 成功報告だけではなく、迷ったこと、難しかったこと、まだ整っていないことも正直に書く。
- 読者へ語りかけるように書く。
- 専門用語を使う場合は、先に感情や状況を置いてから説明する。

## 読みやすさ

- 行間を広めにする。
- 1段落は短くする。
- 2〜4文ごとに空行を入れる。
- スマホで読んだときに、文章の塊が重く見えないようにする。

## 構成

1. 読者が入りやすい感情や場面から始める。
2. 今回の制作・更新内容を伝える。
3. なぜそれをしたのか、背景を書く。
4. まだ課題がある場合は正直に書く。
5. 最後は、次に見てもらいたいものや、制作を続ける意思で締める。

## 禁止

- エラーログや作業ログをそのまま記事にしない。
- 読者に関係のない内部手順を長く書かない。
- 画像や見出しだけで雰囲気を作り、本文が薄い記事にしない。
- 行間のない長文を作らない。
